# 2025 GPS개론 공통주제 프로젝트 8조 코드

## 과제 내용

 - 인하대학교 농구장을 대상으로 **안드로이드 어플**과 **RTAP2U 수신기** 두 개를 이용해 꼭지점 별로 측정합니다.
 - 각 꼭지점은 30초 이상 측정하고, 이때 수집한 NMEA데이터로 분석을 진행합니다.
 - 아이폰은 사용자가 가만히 있을 때 측정을 멈추기 때문에, 스마트폰을 사용할 때는 가급적 안드로이드를 사용해야 합니다.
 - RTAP2U를 사용할 때는 VRS와 연결하여 **FIX** 상태인지 확인하며 사용합니다. (VRS는 국내 측량 전문가들도 모두 사용하기 때문에 사용자가 몰릴 경우 연결이 끊길 수 있습니다)
 - 두 가지 방법을 통해 얻은 평균 좌표를 이용해 각각 너비, 높이, 면적을 계산합니다.
 - 두 가지 방법을 통해 얻은 데이터를 바탕으로 통계적 특성을 고찰합니다.
 - 취득한 VRS 좌표를 참값으로 두고, 각 꼭짓점에서 스마트폰의 측위오차를 분석합니다.

## 아키텍쳐

**현재 진행 단계**
'''
NMEA 읽고 파싱하기[완료] → 위경도toECEF 로직 구현[완료] → 평균, 변위 계산[완료]
    → 점간 평균 거리 및 면적 계산[완료] → 꼭지점 별 통계[진행중] → 오차 분석[대기]
'''

### 데이터구조

 - NMEA: GNGGA(GNSS), GPGGA(GPS)만 사용
 - 추출 정보: 위도, 경도, 고도, 해발고도
 - 좌표계 WGS84(기본) → ECEF 직교좌표로

### 파일 설명

 - phone_data/**: 스마트폰 NMEA 데이터(*.txt)
 - rtk_data/**: RTAP2U VRS FIX 데이터(*.txt)
 - gongtong1_v2.py: NMEA 파싱 + 위경도toECEF + 평균좌표/변위 계산 + csv 저장
 - gongtong2_v2.py: 통계 분석(거리, 면적, 표준편차, 오차 등)
 - output.csv: 계산 결과 산출물
